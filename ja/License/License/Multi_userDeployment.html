はじめにお読みください

Originを配布する方法

OriginLabは、2種類のインストーラを提供しています。

  • InstallShield セットアップ - DVD、体験版のダウンロード、ご購入後に電子メールでお送りするダウンロードリンクから入手できます。（注：複数ユーザーに展開したい場合は、exe形式ではなくzip形式のダウンロードを選択してください。）
  • MSI インストーラ - こちらからダウンロードしてください。

InstallShield セットアップの利点

InstallShield セットアップの利点は、必要なMicrosoft（MS）DLLがコンピュータに存在するかどうかを確認し、存在しない場合は、InstallShield セットアップが必要なMSインストーラを自動的に実行してDLLをインストールすることです。
MSIとの違い - OriginのMSI インストーラは、MSインストーラを自動的に実行できません。したがって、MSIからOriginを展開する場合は、MSインストーラとそれらをインストールするためのバッチファイルを含めることをお勧めします。

MSI インストーラの利点

グループポリシー（GPO）を使用してOriginを展開する場合、MSIインストーラのみを選択でき、InstallShieldセットアップは選択できません。
MSIインストーラを使用すると、ユーザがOriginをサイレントインストールできるようにスクリプトを準備できます。.iniファイルは、シリアル番号、インストールフォルダ、ユーザーファイルフォルダ（ユーザは最初の起動時にこれを選択するように求められます）などを事前設定するために使用されます。同時起動版の場合、FLEXnetサーバーを事前設定することもできます。
InstallShieldセットアップでは.iniファイルを使用できますが、MSIインストーラは複数ユーザへの展開に適したタイプです。これにより、管理者は別の「変換」（MSTファイル）を使用して、より多くの構成を事前設定したり、MSIインストールデータベースのデータを置き換えたりすることができます。変換の詳細については、こちらをご覧ください。

次の手順は、InstallShieldセットアップとMSIインストーラの両方に適用されます。

設定をカスタマイズしたインストーラの用意

Config.iniを使用したインストール設定のカスタマイズ

Config.iniとは何か、Config.ini でできること

Config.iniファイルは、インストール設定を事前設定/カスタマイズするための構成ファイルです。Origin 2019以降のバージョンで使用可能です。InstallShield セットアップの場合、Originセットアップファイル（zip形式のインストーラ）に付属しています。MSIインストーラの場合、個別にダウンロードできます。

Config.iniとは

Config.iniファイルで次の設定を行うことは任意です。表示および変更するためにメモ帳エディタで開くことができます。各設定を理解するのに役立つ内容が含まれています。

インストール設定

[Setup]セクションですべてのインストールデータを事前設定します。

- シリアル番号、ユーザー名、所属情報、インストールパス、PythonコンソールとSPSSインポートを有効にするかどうかを事前設定
- Originをサイレントインストールするかどうかを決定します（注：これはInstallShieldセットアップにのみ適用されます。MSIをサイレントインストールするには、/quietスイッチを使用してコマンドラインを実行できます。）

Note: OriginをサイレントモードでインストールするためにQuietMode = 1に設定した場合、以下の項目を事前設定する必要があります。 

AcceptOriginLabLicenceAgreement=1
SN=
UserName=
Company=

指定しない場合、その他の項目は次のデフォルト値を使用します。 

InstallPath=C:\Program Files\OriginLab\Origin####
EmbeddedPython=1
SPSSImport=0
AllUsers=1
ProgramFolder=C:\ProgramData\Microsoft\Windows\Start Menu\Programs\OriginLab Origin ####
Originフォルダのパス
  • ユーザファイルフォルダ
- ファイルフォルダは[Locations]セクションで事前設定
- Originを最初に起動するときに、事前設定されたパスでユーザファイルフォルダダイアログを表示するかどうかを決定
- ユーザファイルフォルダをOneDrive上に設定すると問題を引き起こす可能性があるため、推奨しません。
  • 自動保存、バックアップ、および未保存のプロジェクトフォルダ
- 自動保存フォルダ(AutoSave)、バックアッププロジェクトフォルダ(Backup)、未保存プロジェクトフォルダ(Unsaved)のルートフォルダを[Locations]セクションで事前設定
  • アプリ
- アプリフォルダ(Apps)を[Apps]セクションで事前設定

Note: Config.iniに事前設定されたフォルダパス（ユーザファイルや自動保存フォルダなど）がある場合は、以前のバージョンの設定を継承する代わりに、それらのパスが使用されます。

    サーバポート番号を61616に設定している場合は、次のように[License]セクションを編集します。
    Originが以前にインストールされている場合、Replaceオプションを設定して現在のサーバー情報を更新するかどうかも決定できます。（注：Replaceオプションは、InstallShieldセットアップ config.iniファイルでのみ使用でき、MSIインストーラでは使用できません。）
  • ネットワーク接続が不安定または低速な場合などで、Originが同時起動ライセンスサーバーからの応答を待機する最大時間を増やす必要があるときは、config.iniの[License] セクションにシステム環境変数FLEXLM_TIMEOUT (マイクロ秒単位) を事前設定します。たとえば、FLEXLM_TIMEOUTを5秒に設定する場合 
    [License] 
FLEXLM_TIMEOUT = 5000000


    FLEXLM_TIMEOUTのデフォルト値は3000000 (3秒)です。より大きな値が既に存在する場合は、以下のように強制で設定しない限り更新されません。 

    FLEXLM_TIMEOUT_FORCE=1

    詳細はこのFAQを参照してください。

オプションの手順

プロダクトキーの事前設定

マルチシート版ライセンスをお持ちで、エンドユーザのプロダクトキーを事前設定して、Originを自動的にアクティブ化できるようにする場合は、グループプロダクトキーをリクエストして、インストーラファイルが配置されているフォルダ内のpk.txtファイルに保存できます。

インターフェイス設定、テンプレート、フィット関数などのカスタマイズファイルを設定する。

メニューやボタン、グラフテンプレート、フィット関数などのエンドユーザのOriginを事前にカスタマイズしたい場合は、インストーラファイルが配置されているフォルダにUpdateUserFilesフォルダを作成し、カスタマイズしたファイル（OMC、XML、 ogwu、fdfなど）をUpdateUserFilesフォルダに移動します。これらは、Originを初めて起動したときに、各ユーザのユーザファイルフォルダにコピーされます。

アプリのプリインストール

アプリをプリインストールしたい場合は、以下の方法があります。
  • Origin セットアップフォルダ（Setup.exeまたはMSIを含む）にAuto Installという名前のフォルダを作成
  • アプリファイル(*.opx)をダウンロードし、Auto Installフォルダに保存
エンドユーザがOriginセットアップフォルダからOriginをインストールすると、 Auto Install フォルダがエンドユーザのOriginプログラムフォルダにコピーされます。Originを起動すると、含まれているアプリ(*.opx) が自動的にインストールされます。

レジストリに保存されたシステム変数を事前設定する

Origin セットアップフォルダ（Setup.exeまたはMSIを含む）のConfig.iniを、メモ帳で開きます。[SysVar]セクションを見つけて、ここに事前設定するシステム変数を追加します。Originをインストールすると、指定したシステム変数が設定されます。

Originインストールパッケージの配布

  1. Originセットアップファイルをコンピュータ「A」のフォルダにコピーします。Note：コンピュータ「A」は、すべてのOriginユーザが接続できるネットワーク上に存在する必要があります。
  2. その変更されたConfig.ini、pk.txtファイルおよびUpdateUserFilesフォルダ（ある場合）が、setup.exeがあるOrigin Setupフォルダにあるのかを確認します。
  3. のフォルダーを完全な読み取り/書き込み権限で共有し、UNC共有名をメモします。(例： \\A\Origin\)
  4. すべてのユーザにメールを送信して、setup.exe/msiファイルの実行を依頼します。ユーザは、管理者特権を持つログインアカウントを使用してコンピュータにログインする必要があります。また、ユーザのコンピュータは\\A\<Origin Setup folder>にアクセスできるネットワーク上に存在する必要があります。

Note:

  • このインストール方法では、ターゲットコンピュータにのみOriginがインストールされます。
  • ネットワーク同時起動型では、登録はオプションです。ユーザがOriginを登録したい場合は、メニューヘルプ:オンライン登録を選択して登録ダイアログを開いて登録できます。

グループフォルダによるインストール後の更新

グループフォルダは、Originがファイルを配布するために提供するオプション機能です。複数のOriginユーザがいるサイト/組織で使用できます。どのライセンスタイプにも対応しています。グループリーダーとグループメンバーのライセンスタイプが異なる場合でもグループフォルダを使用してそれらの間でファイルを共有できます。

グループフォルダ機能には、1つまたは複数のグループリーダーの設定が含まれます。たとえば、グループリーダーは組織の部門レベルで確立できます。組織内のOriginおよびOriginProユーザーのすべてまたは任意のユーザが、同じグループリーダーまたはリーダーを共有できます。リーダーはグループのメンバーへ、カスタマイズしたOriginファイルとパッチをソースとして配布することができます。

グループリーダーを設定したら、組織内のオリジンを設定して、グループリーダーからの更新を受け取るように事前定義されているようにすることができます。

グループリーダーとグループメンバーの設定

最初のステップは グループリーダーマシンのグループフォルダを設定することです。.

次に、 メンバーマシンのグループフォルダを設定します。.

グループフォルダマネージャを使用したファイルの公開

グループリーダーは、グループフォルダマネージャーを、追加、アップデート、またはグループフォルダからファイルを削除、グループのメンバーに公開するといったことを行うために使います。このツールの使い方の詳細についてはグループフォルダマネージャのドキュメントをご参照ください。

グループメンバーがグループフォルダを指定した後、Originの起動時に、グループフォルダに新しいファイルが存在する場合、ファイルが更新されたことを通知するメッセージが表示されます（サポートされているファイルのみで、パッチは対象外です）。あるいは、アップデートが利用可能で、アップデートを受信するためにOriginの再起動が必要であることを知らせるメッセージが表示されます（パッチ用）。

共有グループフォルダからサポートされているファイルは、ユーザーファイルフォルダにあるグループメンバーのフォルダにコピーされます \GroupShared\ 。

ファイルは、テーマオーガナイザ、フィット関数オーガナイザなどのさまざまなOriginダイアログで利用可能になります。該当のファイルには、グループフォルダから作成されたものであることを識別するために、（Group）という先頭タグが付きます。

公開できるファイルの種類

  • Originでサポートしているファイル：プロジェクト、テンプレート、フィット関数などです。
    グループフォルダマネージャを使用してグループリーダマシンのUser Filesフォルダ内のファイルを公開します。
  • アプリ:
    1. グループリーダマシンのUser Filesフォルダに、Auto Installという名前のフォルダを作成
    2. アプリファイル(*.opx)をダウンロードし、Auto Installフォルダに保存
    3. グループフォルダマネージャを使用してアプリファイル（*.opx）を配布
    4. OriginグループメンバーがOriginを開始すると、Originにインストールされているアプリが表示されます。
  • Originのサービスリリースを公開
    OriginLabは、リリースバージョンで見つかった問題を修正するためにサービスリリースを公開することがあります。通常、サービスリリースはパッチとして提供されます。
    1. 最初にグループリーダーマシンにパッチを適用する必要があります。完了したら、グループリーダーマシンを再起動します。
    2. グループフォルダマネージャを使用して グループリーダに適用したパッチファイルを公開します。
    3. OriginグループメンバーがOriginを開始すると、グループリーダーからパッチが入手可能であることを示すメッセージが表示されます。パッチを適用させることができ、Originを閉じると、パッチプログラムの実行が開始されます。
  • FLEXnetサーバを移行し、新しい場所のメンバーを更新する（同時起動型）
    FLEXnetサービスを別のPCに移動した場合、グループフォルダ機能を使用して、この新しい場所で同時起動ネットワークパッケージのメンバーを更新できます。
    1. 新しいFLEXnetサーバをセットアップし、サービスを正常に実行した後、グループリーダーのライセンスフォルダにあるUSE_SERVER.licファイルを編集できます。フォルダの場所:
      C:\ProgramData\OriginLab\License
      FLEXnetサーバ名を<FLEXnet server>とすると、このファイルには通常、次のテキストが含まれます。 
      SERVER <FLEXnet server> ANY
USE_SERVER

      FLEXnetサーバの3台冗長構成もサポートされています。たとえば、3つのサーバ<one>、<two>、<three>で、ポート12345、23456、 34567の場合、USE_SERVER.licファイルは次のようになります。 

      SERVER <one> ANY 12345
SERVER <two> ANY 23456
SERVER <three> ANY 34567
USE_SERVER
    2. 変更が行われ、OriginグループリーダーのPCが起動されると、このファイルは自動的にグループフォルダに公開されます。FLEXnetサーバーの変更がOriginグループメンバーに通知され、「同時ライセンスサーバー設定が更新されました。OKをクリックして再起動してください」というメッセージが表示されます。
